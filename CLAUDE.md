# 감사조서 자동화 앱 개발 가이드라인

본 문서는 프로젝트 개발 시 모든 코드 작성에 반영되어야 하는 핵심 가이드라인입니다.

## 1. 프로젝트 구조 및 관리
- **회사별 격리**: 반드시 회사별로 독립된 폴더(예: `/Company_A`, `/Company_B`)를 생성하고, 각 폴더 내에 해당 회사의 전용 템플릿과 설정 파일(`config.js`)을 위치시킨다.
- **상대 경로 금지**: 파일 읽기/쓰기 시 실행 위치에 따라 에러가 나지 않도록 반드시 `path.join(__dirname, ...)`를 사용하여 절대 경로를 생성한다.

## 2. Playwright 자동화 전략
- **다운로드 이벤트 우선**: 화면 스크래핑보다 앱 내의 '엑셀 다운로드' 버튼 클릭 이벤트를 우선적으로 활용한다.
- **비동기 처리(Wait)**: 데이터 로딩 속도에 따라 버튼이 늦게 뜰 수 있으므로, `waitForSelector`나 `waitForLoadState`를 사용하여 안정적으로 대기한 뒤 동작한다.
- **헤드리스 모드(Headless)**: 개발 및 디버깅 단계에서는 `headless: false`로 설정하여 브라우저 동작을 직접 확인하고, 배포 시에만 `true`로 전환한다.

## 3. 데이터 보안 및 정합성
- **개인정보/기밀 보호**: 로컬 앱 접속 시 사용하는 계정 정보는 코드에 직접 적지 않고 `.env` 파일이나 외부 설정 파일에서 관리한다.
- **금액 데이터 정제**: 엑셀에서 숫자로 인식될 수 있도록 숫자 외의 문자(천 단위 콤마, '원' 표시 등)는 추출 단계에서 제거 후 숫자로 변환(`Number()`)한다.
- **검증 로직 필수**: 데이터 이관 후, 원천 데이터의 합계와 엑셀에 입력된 데이터의 합계가 일치하는지 확인하는 간단한 검증 코드를 포함한다.

## 4. 엑셀 핸들링 (ExcelJS)
- **템플릿 보존**: 원본 템플릿(`template.xlsx`)을 직접 수정하지 않고, 항상 사본을 생성하거나 새로운 이름으로 `writeFile`을 수행한다.
- **서식 유지**: 데이터를 채울 때 기존 템플릿의 셀 테두리나 배경색 등 스타일이 깨지지 않도록 `getRow().values` 방식을 적절히 사용한다.

##5. 엔진과 데이터분리
- 모든 실행 로직은 특정 회사에 종속되지 않아야 하며, 시나리오는 반드시 외부 엑셀(task_list.xlsx)에서 읽어온다.

##6. 대기시간준수
- 화면 전환 및 팝업 오픈 시에는 최소 1초 이상의 안정화 대기 시간을 갖는다
## 5. 분개장 분석(AI-Analysis) 특화 규칙
- **호스트 구분**: 시트명이 `HOST2_`로 시작하거나 분개장 분석 시나리오일 경우 `http://localhost:8081/ai-analysis`로 접속한다.
- **업로드 헬퍼 사용**: 파일 업로드는 반드시 `uploadFileToZone` 헬퍼 함수를 사용하여 안정성을 확보한다.
    - `areaIndex=0`: 분개장 파일 업로드 영역
    - `areaIndex=1`: 계정별원장 파일 업로드 영역
- **로딩 상태 확인**: 업로드 후 즉시 다음 동작을 수행하지 않고, 화면에 "데이터 건수" 또는 "시트 수" 텍스트가 나타날 때까지 `waitForSelector`로 대기한다.
- **데이터 추출 우선순위**: 각 분석 상세 페이지 진입 시, 텍스트 스크래핑보다 '엑셀 다운로드' 버튼 클릭을 통한 원천 데이터 확보를 우선한다.

## 6. 분개장 AI 분석 세션 관리 (연속 처리 핵심 규칙)
- **[초기화면으로] 버튼 필수**: 하나의 분석 메뉴 작업이 끝나면 반드시 화면 우측 상단의 [초기화면으로] 버튼을 클릭하여 대시보드로 복귀한다. **절대로** 브라우저 뒤로가기(`page.goBack()`)나 URL 재접속(`page.goto(url)`)을 사용하지 않는다 — 이 경우 업로드된 분개장 데이터가 소실된다.
- **연속 처리 흐름**: HOST2_ 시트가 여러 개일 때 엄격히 이 순서를 준수한다.
  `파일 업로드 → 메뉴 A 분석/추출 → [초기화면으로] → 메뉴 B 분석/추출 → [초기화면으로] → ...`
- **업로드 생략 (세션 유지)**: [초기화면으로] 복귀에 성공한 경우(`aiSessionActive = true`), 다음 HOST2_ 메뉴에서는 파일 업로드 단계를 건너뛴다.
- **세션 끊김 재시도**: [초기화면으로] 버튼을 찾지 못하거나 대시보드 복귀에 실패하면 세션이 끊긴 것으로 간주(`aiSessionActive = false`)하고, 다음 HOST2_ 메뉴 진입 시 파일을 재업로드한다.
- **구현 위치**: `returnToAiDashboard()` 헬퍼 + `runAudit()` 내 `aiSessionActive` 플래그로 관리된다.
